分步详解

01

明确分析对象与目标

启动

确定要分析哪些产品、分析目的是什么

  • ▸确认参与分析的竞品数量（建议 2-3 个）和被分析产品的类型（B端SaaS / C端产品）
  • ▸明确分析驱动的目标：是为了视觉升级？产品重构？导航优化？
  • ▸加载主框架技能 ux-five-elements-competitor-analysis，按五要素框架启动
ux-five-elements-competitor-analysis
02

战略层 & 范围层分析

抽象层

产品定位 · 目标用户 · 商业模式 · 功能范围

  • ▸通过官网、定价页、文档整理各产品的核心定位、目标用户、商业模式
  • ▸强制：每个产品生成唯一一张功能架构图，调用 product-function-map 技能
  • ▸功能架构图为 HTML 输出时使用纯 CSS 树形图方案（参见 references/html-implementation.md）
  • ▸图后补充战略层&范围层横向对比表格，总结定位差异与壁垒

03

结构层 & 框架层分析

中间层

信息架构 · 导航框架 · 任务动线 · 事实对比 + 原因推论

  • ▸收集竞品真实功能界面截图（Dashboard、列表页、详情页等），调用 b2b-ui-collector
  • ▸对截图进行热区标注（导航区、筛选区、主任务区、结果区），调用 competitor-ui-annotation
  • ▸提取并对比：导航框架（顶导+侧导 / 纯侧导）、页面布局模式（T字型 / 口字型）
  • ▸梳理核心任务完整链路（如：告警触发→排障→根因定位的动线步数）
  • ▸洞察引导：每条事实描述后继续追问「为什么这样设计」——是服务特定用户群？还是为了缩短关键任务路径？

🔍 热区标注规范（competitor-ui-annotation 技能输出）

标注区域

导航区 · 搜索/筛选区 · 主任务区 · 批量操作区 · 结果展示区

坐标格式

top/left/width/height 用百分比表示，支持右侧抽屉叠加场景

⚠️ 表现层截图不做热区标注，避免干扰视觉观察

ux-five-elements-competitor-analysis b2b-ui-collector competitor-ui-annotation
04

表现层视觉分析

具体层

字 · 形 · 色 · 构 · 质 · 视觉选择背后的原因

  • ▸同样调用 b2b-ui-collector 收集表现层截图（日志页、Dashboard、配置页），不做标注
  • ▸从「字 · 形 · 色 · 构 · 质」5个维度逐一拆解，每个维度输出关键数值或特征描述
  • ▸提炼整体视觉风格关键词（如：极客感 / 商务极简 / 宽松舒缓）
  • ▸洞察引导：尝试解读「为什么是这个字号 / 间距 / 色调」——它通常对应着目标用户、品牌定位或信息密度策略

05

综合结论 & 报告交付

输出

竞品优势 · 本品已有优势 · Action Items · HTML 报告

  • ▸汇总五层分析，提炼各竞品核心优势（结构化列表）
  • ▸必须补充：同步总结「本品已有优势」（至少 1 条），避免报告变成单向挑毛病
  • ▸针对「我方产品」输出交互优化建议 + 视觉升级建议（具体可落地）
  • ▸整合所有内容，生成单页 HTML 报告（Tailwind CSS，含响应式适配）
  • ▸本地预览确认后，通过 GitHub API 上传到 GitHub Pages 发布分享链接

技能工具链全景

🧭

ux-five-elements
-competitor-analysis

主框架技能。定义五层分析结构，协调所有子技能，生成最终报告骨架。

角色：指挥官

🗺️

product-function-map

功能架构图技能。每产品生成唯一一张图，清晰展示功能范围，支持 Mermaid 和纯 CSS 两种渲染。

角色：范围层专家

📸

b2b-ui-collector

B端界面截图收集。自动搜索筛选真实功能界面（非营销图），为结构层和表现层提供图源。

角色：素材官

📍

competitor-ui-annotation

界面热区标注。对截图标出导航区/筛选区/主任务区等重点区域，仅用于结构层&框架层分析。

角色：结构层标注师

🚀

github-deploy-publisher

GitHub 发布技能。通过 API 将 HTML 报告推送到 GitHub Pages，无需本地 git 环境。

角色：发布官

📋

five-elements-template.md

标准 Markdown 输出模板。每层都有对应占位结构，可直接复用生成文字版报告。

角色：模板参考

📌 关键分工规则（执行时必须遵守）

战略&范围层 每个产品有且仅有一张功能架构图，调用 product-function-map 生成，不可用文字列表代替
结构&框架层 截图必须热区标注，调用 competitor-ui-annotation；同时每条事实后都要继续追问「为什么这样设计」
表现层 截图不做热区标注，只提供真实原始界面；并输出最急需改善的视觉维度
综合结论 必须同时包含竞品核心优势 + 本品已有优势（至少1条）+ Action Items，保证分析客观完整
发布 本地预览确认后再推送 GitHub，通过 Python + GitHub API 上传（无需本地 git 环境）
